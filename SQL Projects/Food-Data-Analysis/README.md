# Food Sales Data Analysis (SQL)

## 📌 Project Overview
This project leverages relational database queries to analyze corporate food sales data and extract actionable business intelligence. By querying transactional records, the analysis answers 11 critical operational questions regarding regional performance, product-level demand, and seasonal revenue trends.

---

## 🗄️ Technical SQL Concepts Demonstrated
*   **Data Aggregation:** Utilizing `SUM()`, `AVG()`, and `COUNT()` alongside complex `GROUP BY` and `ORDER BY` modifiers to evaluate regional sales performance.
*   **Subqueries & Nested Selects:** Implementing nested logical queries to isolate maximum values across multi-level categorical groups (e.g., identifying top-selling items by individual city).
*   **Time-Series & Conditional Filtering:** Extracting chronological trends using specific date-range logic and aggregate query thresholds (such as Q4 performance analysis).

---

## 📊 Key Business Insights Discovered
*   **City & Regional Metrics:** Isolated exactly which cities generate the highest total revenue to help target regional marketing spend and resource allocation.
*   **Sales Efficiency:** Calculated the Price-to-Quantity ratio across different categories to evaluate pricing power and sales margins.
*   **Product Demand Tracking:** Identified the highest-volume products per city to support hyper-localized inventory and supply chain optimization.
